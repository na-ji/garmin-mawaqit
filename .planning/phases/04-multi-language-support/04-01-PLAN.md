## Trust Boundaries

| Boundary | Description |
|----------|-------------|

| Phone app settings to device | Mosque slug entered on phone, stored via Properties API -- already validated in Phase 1 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | resources-fre/strings/strings.xml | accept | String resources are compiled into the PRG binary at build time. No runtime file access. Tampering would require modifying the built binary on the watch, which is not a realistic attack vector for a Garmin widget. |
| T-04-02 | Information Disclosure | String content in resources | accept | All strings are UI labels with zero sensitive data ("dans", "maintenant", setting labels). No PII, no secrets. |
| T-04-03 | Denial of Service | loadResource() in Glance 28KB budget | mitigate | Research verified ~400 bytes overhead (1.4% of 28KB). Acceptance criteria verify loadResource is called only for needed strings (6 in glance), not bulk-loaded. If memory exceeds budget, CIQ runtime shows "Out of Memory" -- recoverable by restarting widget. |
</threat_model>
