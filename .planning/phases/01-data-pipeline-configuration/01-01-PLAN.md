**onSettingsChanged() as Void:** (implements CONF-02)
- Read new slug: `var newSlug = getMosqueSlug();`
- Compare with `_currentSlug`:
  - If different AND newSlug is not null:
    - Update `_currentSlug = newSlug;`
    - Clear all cached prayer data from Storage by calling `clearCachedData()`
    - Add comment: `// TODO: Plan 02 will trigger fetchPrayerData(newSlug) here`
  - If newSlug is null (user cleared the slug):
    - Set `_currentSlug = null;`
    - Call `clearCachedData()`
- Always call `WatchUi.requestUpdate()` at the end

**Helper: getMosqueSlug() as String or Null:**
- Read from Properties: `var slug = Properties.getValue("mosqueSetting") as String or Null;`
- Per Pitfall 6: Check BOTH null AND empty string: `if (slug == null || slug.equals("")) { return null; }`
- Return slug

**Helper: clearCachedData() as Void:**
- Clear all prayer data storage keys. Use the key pattern from Research Pattern 3:

  - `Storage.deleteValue("todayTimes");`
  - `Storage.deleteValue("lastFetchDate");`
  - `Storage.deleteValue("lastFetchSlug");`

| T-01-01 | Tampering | mosqueSetting property | accept | The slug is a simple string identifier used in a URL path. Garmin's settings system handles transport. Slug validation (URL-safe characters) happens at API request time in Plan 02. Risk: low -- malformed slug produces 404, not injection. |
| T-01-02 | Denial of Service | clearCachedData on slug change | accept | Clearing all 26+ storage keys on every slug change is fast (in-memory operation). No amplification vector. |
| T-01-03 | Information Disclosure | Cached prayer data in Storage | accept | Prayer times are public data (available on mawaqit.net). No PII stored. No sensitive data at risk. |
