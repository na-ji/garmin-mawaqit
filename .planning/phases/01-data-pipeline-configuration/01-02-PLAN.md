<objective>
Implement the data fetching service and storage layer for prayer data and mosque metadata from the Mawaqit proxy API.

Purpose: This is the core data pipeline -- fetching prayer times, calendar data, iqama offsets, and mosque metadata via per-month and dedicated API endpoints, then storing them in Application.Storage for offline use. This plan delivers DATA-01 (fetch from API), DATA-02 (cache in Storage), DATA-03 (offline fallback), DATA-04 (two months for rollover), and DATA-05 (iqama times). It also implements the metadata fetch per D-01/D-08, storing mosque name, timezone, jumua, shuruq, and hijriAdjustment.

Output: Working MawaqitService (HTTP fetch chain including metadata) and PrayerDataStore (storage layer with metadata read accessor) wired into the AppBase lifecycle.

Create `source/MawaqitService.mc` -- the HTTP service that fetches prayer data and mosque metadata through sequential API calls.

**Imports:**
```monkeyc
using Toybox.Communications;
using Toybox.Application.Storage;
using Toybox.Time;
using Toybox.Time.Gregorian;
using Toybox.System;
using Toybox.Lang;
```

**Module or Class: MawaqitService**

Use a module (not a class) since we need static/singleton behavior and Monkey C modules are simpler for this. All functions and variables are module-level.

**Constants:**
```monkeyc
const API_BASE = "https://mawaqit.naj.ovh/api/v1/";
```

**Module-level state variables:**
```monkeyc
var _fetchSlug as String or Null = null;     // Slug being fetched
var _fetchMonth as Number = 0;                // Current month being fetched (1-12)
var _fetchNextMonth as Number = 0;            // Next month to fetch (1-12)
var _isFetching as Boolean = false;           // Prevent concurrent fetches
var _fetchStep as Number = -1;                // Tracks position in fetch chain
```

**Function: fetchPrayerData(slug as String) as Void**
This is the entry point. It kicks off the sequential request chain.
- If `_isFetching` is true, return early (prevent concurrent fetch chains)
- Set `_isFetching = true`
- Set `_fetchSlug = slug`
- Calculate current month: use `Gregorian.info(Time.now(), Time.FORMAT_SHORT)` to get `.month` (1-12)
- Set `_fetchMonth = info.month`
- Set `_fetchNextMonth = (info.month % 12) + 1` (wraps December->January)
- Set `_fetchStep = -1`
- Start chain by calling `_processNextStep()`

**Function: _processNextStep() as Void** (private)
- Increment `_fetchStep`
- Based on `_fetchStep`:
  - 0: `_fetchCalendar(_fetchMonth)` -- current month calendar
  - 1: `_fetchCalendar(_fetchNextMonth)` -- next month calendar
  - 2: `_fetchIqama(_fetchMonth)` -- current month iqama
  - 3: `_fetchIqama(_fetchNextMonth)` -- next month iqama
  - 4: `_fetchMetadata()` -- mosque metadata (per D-08)
  - 5: `_fetchPrayerTimes()` -- today's times
  - 6 or higher: fetch complete, set `_isFetching = false`, record fetch metadata

**Function: _fetchCalendar(month as Number) as Void** (private)
- Build URL: `API_BASE + _fetchSlug + "/calendar/" + month`
- Call `Communications.makeWebRequest(url, null, {:method => Communications.HTTP_REQUEST_METHOD_GET, :responseType => Communications.HTTP_RESPONSE_CONTENT_TYPE_JSON}, method(:onCalendarReceive))`

**Function: onCalendarReceive(responseCode as Number, data) as Void**
- If `responseCode == 200` and `data != null`:
  - Determine which month was fetched based on `_fetchStep` (step 0 = `_fetchMonth`, step 1 = `_fetchNextMonth`)
  - `Storage.setValue("cal_" + month, data)` where month is the numeric month
  - Call `_processNextStep()`
- Else:
  - Set `_isFetching = false` (abort chain silently, per D-06)

**Function: _fetchIqama(month as Number) as Void** (private)
- Build URL: `API_BASE + _fetchSlug + "/calendar-iqama/" + month`
- Call `Communications.makeWebRequest(url, null, {:method => Communications.HTTP_REQUEST_METHOD_GET, :responseType => Communications.HTTP_RESPONSE_CONTENT_TYPE_JSON}, method(:onIqamaReceive))`

**Function: onIqamaReceive(responseCode as Number, data) as Void**
- If `responseCode == 200` and `data != null`:
  - Determine which month based on `_fetchStep` (step 2 = `_fetchMonth`, step 3 = `_fetchNextMonth`)
  - `Storage.setValue("iqama_" + month, data)`
  - Call `_processNextStep()`
- Else:
  - Set `_isFetching = false` (abort chain silently)

**Function: _fetchMetadata() as Void** (private, implements D-08)
- Build URL: `API_BASE + _fetchSlug + "/metadata"`
- Call `Communications.makeWebRequest(url, null, {:method => Communications.HTTP_REQUEST_METHOD_GET, :responseType => Communications.HTTP_RESPONSE_CONTENT_TYPE_JSON}, method(:onMetadataReceive))`

**Function: onMetadataReceive(responseCode as Number, data) as Void**
- If `responseCode == 200` and `data != null`:
  - `Storage.setValue("mosqueMeta", data)` -- stores the full metadata Dictionary (per D-01: name, timezone, jumua, jumua2, shuruq, hijriAdjustment)
  - Call `_processNextStep()`
- Else:
  - Set `_isFetching = false` (abort chain silently, per D-06)

**Function: _fetchPrayerTimes() as Void** (private)
- Build URL: `API_BASE + _fetchSlug + "/prayer-times"`
- Call `Communications.makeWebRequest(url, null, {:method => Communications.HTTP_REQUEST_METHOD_GET, :responseType => Communications.HTTP_RESPONSE_CONTENT_TYPE_JSON}, method(:onPrayerTimesReceive))`

**Function: onPrayerTimesReceive(responseCode as Number, data) as Void**
- If `responseCode == 200` and `data != null`:
  - `Storage.setValue("todayTimes", data)`
  - Record fetch metadata:
    - Get today's date as string using `Gregorian.info(Time.now(), Time.FORMAT_SHORT)`: format as `year + "-" + month + "-" + day`
    - `Storage.setValue("lastFetchDate", dateString)`
    - `Storage.setValue("lastFetchSlug", _fetchSlug)`
  - Set `_isFetching = false`
  - Call `WatchUi.requestUpdate()` to refresh the display with new data
- Else:
  - Set `_isFetching = false`

**CRITICAL IMPLEMENTATION NOTES:**
- Use String keys ONLY for Storage (never Symbols) per Pitfall 3
- Do NOT attempt to fetch the full `/{slug}/` endpoint -- it's 40.3 KB and will cause -402/-403 errors
- Each per-month endpoint returns ~3 KB, metadata ~200 bytes, safely within device limits
- The `/calendar/{month}` endpoint returns an Array (0-indexed), not a Dictionary
- The `/calendar-iqama/{month}` endpoint returns an Array of offset objects
- The `/metadata` endpoint returns a Dictionary with keys: name, timezone, jumua, jumua2, shuruq, hijriAdjustment (per D-08)
- Error codes to expect: -104 (BLE not connected), -300 (timeout), -402 (too large), 404 (bad slug)
- On ANY error, abort the chain silently and leave existing cached data intact (per D-06)
- Import `using Toybox.WatchUi;` for the requestUpdate call

**Module: PrayerDataStore**

This module provides read accessors for the data that MawaqitService writes. Phase 2 will use these functions to get prayer times for display.

**Function: getCalendarMonth(month as Number) as Array or Null**
- Return `Storage.getValue("cal_" + month) as Array or Null`
- The Array is 0-indexed by (day - 1). Each element is a Dictionary with keys: "fajr", "sunrise", "dohr", "asr", "maghreb", "icha"

**Function: getIqamaMonth(month as Number) as Array or Null**
- Return `Storage.getValue("iqama_" + month) as Array or Null`
- The Array is 0-indexed by (day - 1). Each element is a Dictionary with keys: "fajr", "dohr", "asr", "maghreb", "icha" (values are offset strings like "+10")

**Function: getTodayTimes() as Dictionary or Null**
- Return `Storage.getValue("todayTimes") as Dictionary or Null`
- Dictionary keys: "fajr", "sunrise", "dohr", "asr", "maghreb", "icha" (values are time strings like "05:35")

**Function: getMosqueMeta() as Dictionary or Null**

**Function: getLastFetchDate() as String or Null**

- Format: "YYYY-MM-DD" or similar date string

**Function: getLastFetchSlug() as String or Null**
- Return `Storage.getValue("lastFetchSlug") as String or Null`

**Function: getTodayPrayerTimes() as Dictionary or Null**
This is the primary accessor for Phase 2. It returns today's prayer times from the calendar data.
- Get current date info: `var now = Gregorian.info(Time.now(), Time.FORMAT_SHORT);`
- Get month and day: `var month = now.month; var day = now.day;`
- Try calendar first: `var cal = getCalendarMonth(month);`
- If `cal != null` and `cal.size() >= day`:
  - Return `cal[day - 1] as Dictionary` (0-indexed)
- Else: fall back to `getTodayTimes()` (from /prayer-times endpoint)
- If nothing available: return `null`

**Function: getTomorrowPrayerTimes() as Dictionary or Null**
For Isha-to-Fajr rollover (DATA-04). Returns tomorrow's prayer times.
- Get current date info
- Calculate tomorrow: if `day < daysInMonth` then `day + 1, same month`. If last day of month then `day 1, next month`.
- For same month: `var cal = getCalendarMonth(month); return cal[day] as Dictionary;` (day is 0-indexed for tomorrow since day-1+1 = day)
- For next month: `var cal = getCalendarMonth(nextMonth); return cal[0] as Dictionary;`
- If calendar data not available: return `null`

To calculate days in month, use this approach:
```monkeyc
// Get the first of next month, then subtract one day to find last day of current month
var nextMonthNum = (month % 12) + 1;
var nextYear = (month == 12) ? now.year + 1 : now.year;
var firstOfNext = Gregorian.moment({:year => nextYear, :month => nextMonthNum, :day => 1, :hour => 0, :minute => 0, :second => 0});
var lastOfCurrent = Gregorian.info(new Time.Moment(firstOfNext.value() - 86400), Time.FORMAT_SHORT);
var daysInMonth = lastOfCurrent.day;
```

**Function: getTodayIqama() as Dictionary or Null**
Returns today's iqama offsets from the iqama calendar.
- Same month/day logic as getTodayPrayerTimes but reads from `getIqamaMonth(month)`
- Returns Dictionary with offset strings ("+10", "+5", etc.) or null

**Function: hasCachedData() as Boolean**
Per D-07: Check if we have valid (non-expired) cached data for today.
- Get current date info (month, day)
- Check: `var cal = getCalendarMonth(month);`
- If `cal != null` and `cal.size() >= day`: return `true`
- Else check `getTodayTimes() != null`: return `true` (fallback from /prayer-times)
- Else: return `false` (per D-07, this triggers empty state)

**Function: isMosqueConfigured() as Boolean**
- Convenience check: `return getLastFetchSlug() != null;`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Watch App -> Mawaqit Proxy API | HTTP requests cross from watch (via phone BLE) to external API server |
| API Response -> Application.Storage | Untrusted JSON data from API is stored in device storage |
| Application.Storage -> Display Logic | Cached data is read and used to render prayer times (Phase 2+) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
