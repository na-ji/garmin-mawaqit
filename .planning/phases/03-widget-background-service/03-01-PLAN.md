Create `source/MawaqitWidgetView.mc` as a new file. This is a `WatchUi.View` subclass with direct Dc drawing (no XML layouts), following the same patterns as MawaqitGlanceView.

**Class structure:**
```monkeyc
import Toybox.WatchUi;
import Toybox.Graphics;
import Toybox.Timer;
import Toybox.Lang;

class MawaqitWidgetView extends WatchUi.View {
    var _timer as Timer.Timer or Null = null;
    // ... (implementation below)
}
```

**Timer management** (per D-04, matching Glance pattern from MawaqitGlanceView):
- `onShow()`: Create and start `Timer.Timer` with 1000ms interval (1-second updates for countdown accuracy). Widget has generous 64-128KB memory budget so always use 1-second updates (no need for the Glance's 30s/1s adaptive logic).
- `onHide()`: Stop and null out timer.
- `onTimer()`: Call `WatchUi.requestUpdate()`.

**`onUpdate(dc)` layout** (per D-01, D-02, D-03, D-04, D-05, D-06, D-07):

The screen is round AMOLED, 416x416 to 454x454. Layout from top to bottom:

1. **Empty state checks first** (per D-07):
   - If `!PrayerDataStore.isMosqueConfigured()`: Draw "Mawaqit" centered at h/3 using `Graphics.FONT_MEDIUM`, draw "Set mosque in\nGarmin Connect app" centered at 2*h/3 using `Graphics.FONT_SMALL`. Both white on black. Return early.
   - If `!PrayerDataStore.hasCachedData()` or `todayTimes == null` or state is `"no_data"`: Call `drawEmptyState(dc, w, h)` and return.

2. **Get data:**
   - `var todayTimes = PrayerDataStore.getTodayPrayerTimes();`
   - `var tomorrowTimes = PrayerDataStore.getTomorrowPrayerTimes();`
   - `var result = PrayerLogic.getNextPrayerResult(todayTimes, tomorrowTimes);`
   - `var iqama = PrayerDataStore.getTodayIqama();`
   - `var state = result["state"] as String;`

3. **Layout calculations** (proportional to screen size):
   - `var w = dc.getWidth(); var h = dc.getHeight();`
   - Countdown header area: y center at `h * 15 / 100` (15% from top)
   - Separator line: y at `h * 24 / 100`
   - Prayer rows: 5 rows evenly spaced from `h * 30 / 100` to `h * 82 / 100`
   - Row height calculation: `var rowSpacing = (h * 52) / (100 * 5);` -- each row spaced evenly
   - Row y positions: `var rowStartY = h * 30 / 100; var rowY = rowStartY + (i * rowSpacing) + (rowSpacing / 2);` for i=0..4
   - Horizontal margins: `var leftMargin = w * 12 / 100;` and `var rightMargin = w - (w * 12 / 100);`

4. **Draw countdown header** (per D-02, D-04):
   - Determine countdown text using same logic as Glance:
     - state "now": `PrayerLogic.formatCountdown(0, result["name"] as String)`
     - state "normal": `PrayerLogic.formatCountdown(result["remaining"] as Number, result["name"] as String)`
     - state "overnight": `PrayerLogic.formatCountdown(result["remaining"] as Number, result["name"] as String)`
   - Draw in white using `Graphics.FONT_MEDIUM` centered at `(w/2, h * 15 / 100)` with `TEXT_JUSTIFY_CENTER | TEXT_JUSTIFY_VCENTER`

5. **Draw separator line** (per D-02):
   - `dc.setColor(Graphics.COLOR_DK_GRAY, Graphics.COLOR_TRANSPARENT);`
   - `dc.fillRectangle(leftMargin, h * 24 / 100, w - 2 * leftMargin, 2);` -- thin horizontal line

6. **Draw 5 prayer rows** (per D-01, D-03, D-05):
   - Determine the highlighted index:
     - state "normal": `result["index"] as Number` (0-4, the NEXT prayer)
     - state "overnight": highlight nothing (all prayers have passed; or highlight index 0 for Fajr since that's next -- use index 0 since overnight means Fajr is next)
     - state "now": `result["index"] as Number` (the prayer that just occurred)
     - state "no_data": -1 (no highlight)
   - For the "overnight" state, highlight Fajr (index 0) since that is the next prayer.
   - Loop i from 0 to 4:
     - Get prayer key: `PrayerLogic.PRAYER_KEYS[i]`
     - Get prayer label: `PrayerLogic.PRAYER_LABELS[i]`
     - Get prayer time: `todayTimes[PrayerLogic.PRAYER_KEYS[i]]` as String, or "--:--" if null
     - Get iqama offset: if iqama is not null, `iqama[PrayerLogic.PRAYER_KEYS[i]]` as String (e.g., "+10"), else null
     - Calculate row Y: `rowStartY + (i * rowSpacing) + (rowSpacing / 2)`
     - **If this is the highlighted row** (i == highlightIndex):
       - Draw accent background: `dc.setColor(0x00AA44, Graphics.COLOR_TRANSPARENT);` then `dc.fillRoundedRectangle(leftMargin - 4, rowY - rowSpacing/2 + 2, w - 2*leftMargin + 8, rowSpacing - 4, 8);`
       - Set text color to white: `dc.setColor(Graphics.COLOR_WHITE, Graphics.COLOR_TRANSPARENT);`
       - Use `Graphics.FONT_SMALL` (bolder/larger than XTINY) for highlighted row text
     - **Else** (non-highlighted row):
       - Set text color to light gray: `dc.setColor(Graphics.COLOR_LT_GRAY, Graphics.COLOR_TRANSPARENT);`
       - Use `Graphics.FONT_XTINY` for normal row text
     - Draw prayer label at `(leftMargin, rowY)` left-justified
     - Draw prayer time at `(w/2, rowY)` center-justified
     - If iqama offset is not null and not empty string, draw it at `(rightMargin, rowY)` right-justified using a slightly dimmer color (`Graphics.COLOR_DK_GRAY` for normal rows, `Graphics.COLOR_LT_GRAY` for highlighted row)

7. **`drawEmptyState(dc, w, h)` method** (per D-07):
   - Clear screen black
   - Draw "Mawaqit" title centered at h * 12 / 100 using FONT_MEDIUM, white
   - Draw separator line at h * 22 / 100
   - Draw "-- in --" centered at h * 15 / 100 using FONT_MEDIUM (countdown placeholder)
   - Actually, simpler approach matching the normal layout structure:
     - Countdown: "-- in --" at header position using FONT_MEDIUM, white
     - Separator line
     - 5 rows with prayer labels but "--:--" for times, no iqama, no highlight
     - Use same layout positions as the normal state
   - Loop i from 0 to 4: draw `PrayerLogic.PRAYER_LABELS[i]` at leftMargin, "--:--" at center, no iqama. All in COLOR_DK_GRAY.

**Important implementation notes:**
- Do NOT add `(:glance)` annotation to this class. Widget view runs in the full widget context, not glance context. The (:glance) annotation would needlessly add this class to the 28KB glance memory budget.
- Do NOT use `System.println()` anywhere (memory risk).
- Do NOT import Toybox.Time -- countdown uses PrayerLogic.formatCountdown() which handles everything.
- Do NOT include sunrise/shuruq -- per D-06, only the 5 daily prayers are shown.
